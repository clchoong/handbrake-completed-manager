# HandBrake Completed Manager 0.6.0

Version 0.6.0 adds safe completed-history recovery from HandBrake's retained activity logs and resolves two desktop lifecycle responsiveness problems.

> **Independent third-party project:** This project is not affiliated with, endorsed by, or maintained by the HandBrake project. “HandBrake” is used only to identify compatibility with the official [HandBrake](https://handbrake.fr/) application. The package does not include HandBrake code, binaries, documentation, or artwork.

## Highlights

- Review all `.txt` activity logs in `%APPDATA%\HandBrake\logs` from the new **Import logs** action.
- Recover only logs with successful HandBrake completion markers, valid absolute paths, and an output file that still exists.
- Preserve a missing source's last known path while marking the recovered record **Source Missing**.
- Display paused, failed, malformed, missing-output, and duplicate logs with their skip reason before confirmation.
- Deduplicate against existing completed history and other logs in the same review.
- Revalidate output existence immediately before insertion and never modify HandBrake logs, settings, Queue data, or media files.
- Exit reliably from the notification-area menu without synchronously blocking the WPF interface on diagnostic logging.
- Use a styled exact-path confirmation for single-record history removal and keep SQLite deletion work away from the interface thread.

HandBrake's own documentation confirms that individual Windows encode logs are retained until manually deleted and that activity logs include encoding progress and issues. See [HandBrake Activity Log](https://handbrake.fr/docs/en/latest/help/activity-log.html).

## Download

Download `HandBrake-Completed-Manager-0.6.0-win-x64.zip`, extract the complete folder, and read `PORTABLE-README.txt` before configuring HandBrake.

The package is self-contained for 64-bit Windows and requires neither administrator access nor a separate .NET installation. It targets Windows 10 build 17763 or later and Windows 11. It is not code-signed, so Windows may display a reputation warning.

## Validation

The Release build completed with zero warnings and zero errors. All 221 automated tests passed, together with the completion-receiver integration and packaged desktop startup smoke tests. A read-only scan of the available HandBrake sample directory found 102 logs, parsed all 94 successful completed encodes, and separated 11 existing outputs from 83 missing outputs. Archive inspection found no generated history, settings, logs, private developer metadata, or local development paths.
