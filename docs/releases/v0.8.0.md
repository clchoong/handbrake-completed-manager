# HandBrake Completed Manager 0.8.0

Version 0.8.0 redesigns source replacement around direct, familiar file operations and removes checksum-heavy archival work from the normal path.

## Direct replacement

- **Replace Source** is the default and consumes the separate HandBrake output, like cut and paste.
- **Replace Source and Keep Output** copies the output and retains it.
- Same-volume default replacement uses fast filesystem moves.
- Cross-volume and keep-output transfers show byte progress and can be cancelled before the original is changed.
- Cancelled transfers clean up their partial copy and can be retried immediately.
- The workflow performs a file-length check but does not create a source backup or calculate SHA-256 checksums.
- The warning explicitly states that the original source cannot be recovered.
- Multi-selection supports the same two output-retention choices and processes records sequentially.

## History and windows

- The redundant HandBrake completion status and separate source-replacement columns are consolidated into one Status column.
- Status values are **Output Deleted**, **Source Replaced**, and **Source Replaced, Output Kept**.
- Every custom secondary window uses a software-rendered first frame to prevent transparent or blank popup content on affected Windows graphics configurations.
- Replacement and bulk-removal processing waits for its progress window to render before starting work.

## Compatibility

The package supports 64-bit Windows 10 build 17763 or later and Windows 11. Legacy Recovery remains available for unfinished verified replacement operations created by earlier releases.
