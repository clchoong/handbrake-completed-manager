# HandBrake Completed Manager 0.5.0

Version 0.5.0 adds deliberate multi-record selection and bulk file-management actions to the completed-encode library.

> **Independent third-party project:** This project is not affiliated with, endorsed by, or maintained by the HandBrake project. “HandBrake” is used only to identify compatibility with the official [HandBrake](https://handbrake.fr/) application. The package does not include HandBrake code, binaries, documentation, or artwork.

## Highlights

- Select multiple records with Ctrl-click or Shift-click, use Ctrl+A while the table is focused, or choose **Select all shown** for the current filtered results.
- Replace multiple sources safely after a single review that lists every exact source and target path.
- Block cross-selection conflicts when two records would resolve to the same final path.
- Recycle multiple output files while retaining their source files and history records.
- Remove multiple history records without modifying either source or output files.
- Process records sequentially, continue after an individual failure, stop safely between records, and display succeeded, failed, and skipped totals.
- Keep playback, Explorer reveal, and detailed recovery actions limited to one selected record where their meaning is unambiguous.

Every bulk file action uses the existing per-record verification and recovery boundaries. Managed media retirement uses the Windows Recycle Bin; there is no permanent-delete fallback.

## Download

Download `HandBrake-Completed-Manager-0.5.0-win-x64.zip`, extract the complete folder, and read `PORTABLE-README.txt` before configuring HandBrake.

The package is self-contained for 64-bit Windows and requires neither administrator access nor a separate .NET installation. It targets Windows 10 build 17763 or later and Windows 11. It is not code-signed, so Windows may display a reputation warning.
