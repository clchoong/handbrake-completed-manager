# HandBrake Completed Manager 0.7.1

Version 0.7.1 corrects source replacement for encodes whose source and output use the same file extension and ensures operation windows paint before filesystem or database work begins.

## Fixes

- Same-extension source and output files are no longer rejected merely because the planned final path is the original source path.
- The app copies and SHA-256 verifies the converted file, creates and verifies an original backup, persists recovery intent, and then uses the Windows atomic file-replacement operation.
- The recorded HandBrake output remains unchanged at its output path.
- In-place replacement recovery recognizes whether the original or converted digest currently occupies the shared source/final path.
- Replacement progress starts only after the progress window has rendered.
- Bulk history removal waits for its progress window's first completed render before database work starts, preventing a transparent or apparently frozen dialog.
