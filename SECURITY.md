# Security policy

## Supported version

Security fixes target the latest published release and the current `main` branch.

## Reporting a vulnerability

Please do not open a public issue for a vulnerability that could put users or their files at risk. Use GitHub's **Report a vulnerability** option in the repository Security tab to submit a private security advisory.

Include the affected version, reproduction steps, expected impact, and any relevant paths or logs after removing personal information. Do not attach private media files or a real history database.

This project performs file-management operations. Reports involving path validation, replacement recovery, Recycle Bin behavior, database integrity, or unintended file modification are treated as security-sensitive.
